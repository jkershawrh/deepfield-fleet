== CLI Flags

The fleet-controller binary accepts the following flags:

[cols="2,1,3"]
|===
| Flag | Default | Description

| `--mode`
| `all`
| Deployment mode: `all`, `controller`, `proxy`, or `cost`

| `--backends`
| `[]`
| Comma-separated list of backend model server URLs

| `--auth-secret`
| `""`
| Name of the Kubernetes Secret containing the auth token

| `--max-inflight`
| `100`
| Maximum concurrent in-flight inference requests before load shedding

| `--grpc-port`
| `9090`
| Port for gRPC communication with fleet agents and data plane

| `--http-port`
| `8000`
| Port for the HTTP API server

| `--metrics-port`
| `9091`
| Port for Prometheus metrics endpoint

| `--ledger-endpoint`
| `""`
| ARE Ledger gRPC endpoint (e.g., `ledger:9092`)

| `--log-level`
| `info`
| Log level: `debug`, `info`, `warn`, `error`

| `--kubeconfig`
| `""`
| Path to kubeconfig (uses in-cluster config when empty)
|===

== Environment Variables

[cols="2,1,3"]
|===
| Variable | Default | Description

| `FLEET_AUTH_SECRET_FILE`
| `""`
| Path to file containing the auth token (alternative to `--auth-secret`)

| `FLEET_AUTH_TTL`
| `3600`
| Token TTL in seconds

| `FLEET_URL`
| `""`
| fleet-llm-d API endpoint (used by deepfield-fleet dashboard and CLI tools)

| `FLEET_TOKEN`
| `""`
| Bearer token for API authentication (used by clients)

| `FLEET_LEDGER_ENDPOINT`
| `""`
| ARE Ledger gRPC endpoint (alternative to `--ledger-endpoint`)

| `FLEET_LOG_LEVEL`
| `info`
| Log level (alternative to `--log-level`)

| `FLEET_MAX_INFLIGHT`
| `100`
| Max concurrent requests (alternative to `--max-inflight`)
|===

== Helm Values

Complete reference for `charts/fleet-llm-d/values.yaml`:

[source,yaml]
----
# Deployment mode: all, controller, proxy, cost
mode: all

# Replica count
replicas: 1

# Container image
image:
  repository: quay.io/deepfield-fleet/fleet-controller
  tag: latest
  pullPolicy: Always

# Authentication
auth:
  enabled: false
  secretName: fleet-auth

# Rate limiting
rateLimit:
  enabled: true
  defaultRPM: 60        # Default requests/min when no TenantProfile matches

# Load shedding
loadShedding:
  enabled: true
  maxInflight: 100       # Max concurrent requests before shedding

# Backend model servers
backends: []
# - name: ovms-cpu
#   url: http://ovms:8080
#   type: cpu
# - name: vllm-gpu
#   url: http://vllm:8000
#   type: gpu

# gRPC configuration
grpc:
  port: 9090

# ARE Ledger
ledger:
  enabled: false
  endpoint: ""           # e.g., are-ledger:9092

# Resource requests/limits
resources:
  requests:
    cpu: 500m
    memory: 256Mi
  limits:
    cpu: "2"
    memory: 1Gi

# Service configuration
service:
  type: ClusterIP
  httpPort: 8000
  grpcPort: 9090
  metricsPort: 9091
----

== CRD Field Reference

=== FleetInferencePool

[source,yaml]
----
apiVersion: fleet.llm-d.io/v1alpha1
kind: FleetInferencePool
metadata:
  name: granite-3b
  namespace: fleet-llm-d
spec:
  modelName: granite-3b           # Model identifier
  modelVersion: "1.0"             # Model version
  replicas: 2                     # Desired replica count
  hardware:
    type: cpu                     # cpu or gpu
    accelerator: amx              # amx, cuda, rocm
    gpuType: ""                   # h100, a100, mi300x (when type=gpu)
  serving:
    runtime: ovms                 # ovms, vllm, tgi
    quantization: int8            # fp16, int8, int4
    format: openvino-ir           # openvino-ir, safetensors, gguf
  resources:
    requests:
      cpu: "4"
      memory: "8Gi"
    limits:
      cpu: "8"
      memory: "16Gi"
status:
  ready: true
  availableReplicas: 2
  conditions: []
----

=== FleetScalingPolicy

[source,yaml]
----
apiVersion: fleet.llm-d.io/v1alpha1
kind: FleetScalingPolicy
metadata:
  name: latency-driven
spec:
  targetRef:
    kind: FleetInferencePool
    name: granite-3b
  minReplicas: 1
  maxReplicas: 8
  metrics:
    - type: latency
      target:
        p95: 5000ms               # Scale when P95 exceeds 5s
    - type: queueDepth
      target:
        maxPending: 50            # Scale when queue exceeds 50
    - type: utilization
      target:
        cpu: 80                   # Scale when CPU > 80%
  sloGate:
    enabled: true
    rollbackOnBreach: true        # Auto-rollback if SLO breached after scaling
----

== Event Profile YAML

Event profiles define scheduled events that trigger predictive pre-warming:

[source,yaml]
----
# config/events/summit_connect.yaml
name: summit_connect
description: "Red Hat Summit Connect demo event"
schedule:
  startTime: "2026-06-15T09:00:00Z"
  endTime: "2026-06-15T17:00:00Z"
  preWarmMinutes: 30
expected:
  concurrentUsers: 200
  peakRPS: 50
  burstMultiplier: 4
models:
  - name: granite-350m
    targetReplicas: 2
  - name: granite-3b
    targetReplicas: 4
  - name: granite-8b
    targetReplicas: 2
actions:
  - type: PreWarmIntent
    confidence: 0.90
    requireApproval: false
----

Place event profiles in `config/events/` and they will be loaded by the `event_calendar` nanoagent.
