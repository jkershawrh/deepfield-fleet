== What is fleet-llm-d?

fleet-llm-d is a *fleet-level inference orchestration platform* built on llm-d and Red Hat OpenShift, optimized for Intel Xeon processors.
It turns scattered, single-cluster model serving into a governed, cost-optimized, multi-cluster inference fleet.

The thesis is straightforward:

[quote]
____
GPU inference costs $32/hr per H100.
Intel Xeon 6 with OVMS C++ serves the same models at $0.60/hr.
53x cheaper -- on the hardware you already own.
____

fleet-llm-d does not serve models.
It orchestrates the fleet of model servers -- deciding where models run, how traffic routes, when to scale, who gets what quota, and recording every decision in an immutable audit trail.

== Three-Layer Architecture

The platform is built in three layers, each independently deployable:

[cols="1,2,2,2"]
|===
| Layer | Component | Role | Runtime

| *Actuator*
| fleet-llm-d
| Go control plane (31 endpoints, 7 CRDs) + Rust data plane (cross-cluster gateway)
| Kubernetes operator + gRPC/HTTP services

| *Brain*
| deepfield-fleet
| Predictive intelligence: 4 fleet nanoagents, SLO forecaster, consequence scoper, intent emitter
| Next.js dashboard + Python analytics

| *Ledger*
| ARE Immutable Ledger
| Compliance audit trail: 5 chain types, SHA-256 hash linking, proof receipts
| Independent gRPC service (shared infrastructure)
|===

The actuator manages fleet state through CRDs.
The brain predicts SLO breaches and emits typed intents (PreWarm, Scale, ShedLoad, Alert).
The ledger records every decision for tamper-proof auditability.
