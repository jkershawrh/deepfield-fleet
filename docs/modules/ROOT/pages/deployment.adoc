== fleet-llm-d (Helm Chart)

=== Quick Start

Deploy fleet-llm-d using the Helm chart:

[source,bash,role=execute]
----
helm install fleet-llm-d charts/fleet-llm-d/ \
  --namespace fleet-llm-d \
  --create-namespace \
  --set mode=all \
  --set auth.enabled=true \
  --set auth.secretName=fleet-auth
----

Verify the deployment:

[source,bash,role=execute]
----
curl $FLEET_URL/healthz
----

=== Helm Values

Key values in `charts/fleet-llm-d/values.yaml`:

[cols="2,1,3"]
|===
| Value | Default | Description

| `mode`
| `all`
| Deployment mode: `all` (controller + proxy + cost), `controller`, `proxy`, or `cost`

| `replicas`
| `1`
| Number of fleet-controller replicas

| `image.repository`
| `quay.io/deepfield-fleet/fleet-controller`
| Container image repository

| `image.tag`
| `latest`
| Container image tag

| `auth.enabled`
| `false`
| Enable bearer token authentication

| `auth.secretName`
| `fleet-auth`
| Kubernetes Secret containing the auth token

| `rateLimit.enabled`
| `true`
| Enable per-tenant rate limiting

| `rateLimit.defaultRPM`
| `60`
| Default requests per minute when no TenantProfile matches

| `loadShedding.enabled`
| `true`
| Enable load shedding under overload

| `loadShedding.maxInflight`
| `100`
| Maximum concurrent in-flight requests before shedding

| `backends`
| `[]`
| List of backend model server endpoints

| `grpc.port`
| `9090`
| gRPC port for data plane communication

| `ledger.endpoint`
| `""`
| ARE Ledger gRPC endpoint (e.g., `ledger:9092`)

| `ledger.enabled`
| `false`
| Enable ARE Ledger integration
|===

=== Production Configuration

For production deployments with multiple backends and ledger integration:

[source,bash,role=execute]
----
helm install fleet-llm-d charts/fleet-llm-d/ \
  --namespace fleet-llm-d \
  --create-namespace \
  --set mode=all \
  --set replicas=3 \
  --set auth.enabled=true \
  --set auth.secretName=fleet-auth \
  --set rateLimit.enabled=true \
  --set loadShedding.enabled=true \
  --set loadShedding.maxInflight=200 \
  --set grpc.port=9090 \
  --set ledger.enabled=true \
  --set ledger.endpoint=are-ledger:9092 \
  --set-json 'backends=[{"name":"ovms-cpu","url":"http://ovms:8080","type":"cpu"},{"name":"vllm-gpu","url":"http://vllm:8000","type":"gpu"}]'
----

== deepfield-fleet (Dashboard)

=== Container Deployment

Deploy the deepfield-fleet dashboard container:

[source,bash,role=execute]
----
podman run -d \
  --name deepfield-fleet \
  -p 3000:3000 \
  -e FLEET_URL=http://fleet-llm-d:8000 \
  -e FLEET_TOKEN=$FLEET_TOKEN \
  quay.io/deepfield-fleet/deepfield-fleet:latest
----

=== Environment Variables

[cols="2,1,3"]
|===
| Variable | Required | Description

| `FLEET_URL`
| Yes
| fleet-llm-d API endpoint (e.g., `http://fleet-llm-d:8000`)

| `FLEET_TOKEN`
| Yes
| Bearer token for fleet-llm-d API authentication

| `FLEET_AUTH_SECRET_FILE`
| No
| Path to file containing the auth token (alternative to `FLEET_TOKEN`)

| `FLEET_AUTH_TTL`
| No
| Token TTL in seconds (default: 3600)

| `NEXT_PUBLIC_FLEET_URL`
| No
| Fleet URL for client-side API calls (defaults to `FLEET_URL`)

| `PORT`
| No
| Dashboard listening port (default: 3000)
|===

=== OpenShift Deployment

Deploy on OpenShift with a Route:

[source,bash,role=execute]
----
oc new-project deepfield-fleet

oc create secret generic fleet-secrets \
  --from-literal=FLEET_TOKEN=$FLEET_TOKEN

oc apply -f deploy/deepfield-fleet.yaml
----

The deployment manifest creates:

[cols="1,2"]
|===
| Resource | Purpose

| *Deployment*
| deepfield-fleet container (port 3000)

| *Service*
| Routes traffic to port 3000

| *Route*
| TLS-terminated edge route with redirect for insecure traffic

| *Secret*
| `fleet-secrets` with FLEET_TOKEN
|===

=== OpenShift Route Configuration

Create the route manually if not using the manifest:

[source,bash,role=execute]
----
oc create route edge deepfield-fleet \
  --service=deepfield-fleet \
  --port=3000 \
  --insecure-policy=Redirect
----

== Grafana and Prometheus

=== Prometheus Setup

fleet-llm-d exposes metrics on `/metrics` in Prometheus format.
Key metrics:

[cols="2,3"]
|===
| Metric | Description

| `fleet_inference_requests_total`
| Total inference requests by model, cluster, tenant

| `fleet_inference_latency_seconds`
| Request latency histogram (P50, P95, P99)

| `fleet_scaling_events_total`
| Scaling events by type (hpa, predictive, intent)

| `fleet_cost_dollars`
| Inference cost in dollars by hardware tier

| `fleet_tenant_tokens_used`
| Tokens consumed per tenant

| `fleet_load_shed_total`
| Load shedding events (503 responses)

| `fleet_ledger_entries_total`
| ARE Ledger entries by chain type
|===

If using OpenShift built-in monitoring, create a ServiceMonitor:

[source,bash,role=execute]
----
cat <<'EOF' | oc apply -f -
apiVersion: monitoring.coreos.com/v1
kind: ServiceMonitor
metadata:
  name: fleet-llm-d
  namespace: fleet-llm-d
spec:
  selector:
    matchLabels:
      app: fleet-llm-d
  endpoints:
    - port: http
      path: /metrics
      interval: 15s
EOF
----

=== Grafana Dashboards

Import the fleet dashboards from `deploy/grafana/`:

* `fleet-overview.json` -- cluster health, model metrics, cost summary
* `fleet-inference.json` -- per-model latency, throughput, error rate
* `fleet-governance.json` -- tenant quotas, rate limits, budget alerts
* `fleet-scaling.json` -- replica timeline, intent history, SLO compliance

== Verification

Run the deployment verification script:

[source,bash,role=execute]
----
./deploy/verify.sh
----

The script checks:

[cols="1,3"]
|===
| Category | Checks

| *Infrastructure*
| fleet-llm-d pod running, healthz responds, CRDs installed

| *API*
| Fleet management endpoints, inference proxy, cost engine

| *Governance*
| TenantProfile enforcement, rate limiting, budget alerts

| *Ledger*
| ARE Ledger connection, chain integrity

| *Dashboard*
| deepfield-fleet container running, frontend loads
|===
